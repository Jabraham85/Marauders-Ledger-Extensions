# Deadline Notifications

**Category:** Tools  
**Version:** 1.0.0  
**Author:** WB Games Studio  
**Entry point:** `NotificationsPopover.jsx` (sidebar bell)

---

## Overview

Deadline Notifications adds a bell icon to the sidebar that alerts you to overdue and upcoming task deadlines across all departments. It is a purely presentational extension — it receives its data via props from the Sidebar component rather than fetching or computing anything itself. No configuration required, no data stored.

---

## Features

- **Bell icon with badge** — A bell icon appears in the sidebar. When there are overdue tasks, a red badge shows the count (capped at "9+" for display).
- **Popover panel** — Clicking the bell opens a compact dropdown listing all notifications. Each notification shows:
  - A coloured title: red for "Overdue", amber for "Due Today", blue for upcoming items.
  - The task name and department context.
- **Empty state** — If no alerts exist, a "No alerts" message is shown.
- **Auto-populated** — Notifications are generated by the Sidebar component (in the host frame) which scans all tasks with deadlines and classifies them as overdue, due today, or due soon.

---

## Notification Categories

| Category | Condition |
|----------|-----------|
| Overdue | Task deadline is before today (strictly) |
| Due Today | Task deadline is today's date |
| Upcoming | Task deadline is within the next 3 days |

---

## How It Works

`NotificationsPopover` is a presentational component — it renders whatever it is passed:

```jsx
<NotificationsPopover
  notifications={notifications}          // array of { title, body }
  showNotifications={showNotifications}  // boolean
  onToggleNotifications={toggle}         // callback
/>
```

The `notifications` array and `showNotifications` state are managed by `Sidebar.jsx` in the host frame. The Sidebar computes the notification list from `useStore()` state on every render, classifies tasks by deadline proximity, and passes the result down as props.

This means the Notifications extension is essentially zero-overhead — there is no polling, no timer, no background work.

## Technical Notes

- `NotificationsPopover.jsx` is 48 lines — one of the smallest extensions in the catalogue.
- Because the component is prop-driven, it works identically whether or not the bell is currently shown. The popover visibility is toggled by the parent via the `showNotifications` boolean.
- The overdue count badge clips at 9 (shows "9+" for 10 or more), keeping the badge visually compact.
- Uses `useTheme` / `W` for theming consistency with the rest of the app.

## Installation

1. Open the Marketplace tab in The Marauder's Ledger.
2. Find **Deadline Notifications** and click **Install**.
3. Enable it from the Extensions page.
4. The bell icon will immediately appear in the sidebar and start alerting on overdue tasks.
